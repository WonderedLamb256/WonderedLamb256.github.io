Gentoo .ebuild files: How do you write one?

bibliography: https://wiki.gentoo.org/wiki/Basic_guide_to_write_Gentoo_Ebuilds

So you want to make a Gentoo Linux program? No problem. However, you'll need to create an ebuild file first. An ebuild file is a shell script file with the extension of .ebuild. It contains basic variables and metadata needed to define a package. Ebuilds are stored in /var/db/repos/ and official ebuilds are contained in a folder named gentoo inside of that folder. Assuming you have a package and are using github or gitlab/hosting a git server, fire up your favorite text editor. Comments are pound signs like normal shell scripts, Make, and Python. 

                        
                        # Copyright 1999-2020 Gentoo Authors

                        EAPI=7 # Use Portage API v7
                        
                        DESCRIPTION="My library" # Description for Portage tools
                        HOMEPAGE="https://wonderedlamb256.github.io/" # Your homepage
                        SRC_URI="https://wonderedlamb256.github.io/downloads/blahblahblah-3.1.git" # Download link
                        
                        LICENSE="GPL-3" # This code would be under GPL3 if it existed
                        SLOT="0" # You can use very similar versions of the same package twice or more using slots
                        KEYWORDS="~amd64" # Use tilde to denote testing architecture. i.e. this is in testing for AMD64 only.
                        IUSE="" # This package does not depend on any packages.
                        
                        DEPEND="" # Further dependency info.
                        RDEPEND="${DEPEND}" # Ditto.
                        BDEPEND="" # Ditto.

Now that we have our basic ebuild, we may want to customize the build process. Here's a list of ebuild functions that are involved in the build process.

  • pkg_pretend: (What to do when) pretend installing
  • src_unpack: Unpacking tar.gz file
  • src_prepare: Preparing for compile
  • src_configure: reading config info
  • src_compile: compiling
  • src_test: testing
  • src_install: installation
There are some similarities with normal shell commands, but there are also special actions that exist in Gentoo ebuild functions. Read more about actions
