That‘s 1BADB002 part 2: Dissecting the Master Boot Record

Sector 1. What is the Master Boot Record?

The Master Boot Record is a 512 byte long piece of code at the start of an MBR-formatted hard drive. It serves the purpose of setting up basic routines and calling the bootloader, such as GRUB or Windows Boot Manager. It is unique in that it is not under any filesystem, and is thus not a file.

Sector 2. Making the Master Boot Record a File

In order to hex dump and dissect the Master Boot Record, we'll need some basic tools.

  • dd (from GNU coreutils)
  • A hex dumper or hex editor (we‘ll use hexdump, which I believe is from binutils or coreutils)
In addition, you may want:
  • file (from GNU coreutils)
Now that we have the required tools, you“ll want to run this command on a system that only uses MBR. (If you don“t have a system using MBR/legacy you may need to set up a virtual machine.)
