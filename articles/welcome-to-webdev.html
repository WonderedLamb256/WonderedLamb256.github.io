Lamb's Web Developing Guide pt. 1: Intro to HTML+CSS

Welcome to the wonderful world of web development!
Web development is a fun and relaxing task. However, before you begin to make web pages, you'll need to know how to. “But how?”, you say. That's exactly what you'll be learning today. I'd like to begin with explaining all of the web development languages so you're familiar with them.

Web Development Languages

Web development comes in many different forms - HTML, CSS, JavaScript... I'll give you a short and simple explanation for all of those languages. P.S. a backend language is server side, a frontend language is client side.

  • HTML - The skeleton of the web. Without HTML, you'd only have primordial websites that use backend languages or JavaScript to write.

  • XML - Another markup class language used mainly for SEO and configuration today.

  • CSS - The muscles of the web. This makes a website look advanced. Sometimes used with preprocessors like Sass or LESS or with frameworks like Bootstrap.

  • Sass/SCSS or LESS - CSS with a brain, aka CSS with functions, mixins and other neat things.

  • JavaScript - The brain of the web. Can be improved with libraries like jQuery, and has many features of a typical programming language .

  • PHP - The remaining internal organs of the web. Useful for transferring data to servers and can work like JavaScript.

  • Rust and C++ - Wait, what!? Huh!? WebAssembly!? What!?

  • Python - Not exclusive to web development, but Django and Flask are frequently used as web server frameworks.

  • SQL - Structured Query Language. Is useful for putting data in databases.

  • NoSQL - *ahem* Well... NOT Structured Query Language.

  • Java - Java applets and Spring. That's pretty much all.
